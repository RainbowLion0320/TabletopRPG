# 决策：MVP 功能范围

> **状态：进行中**
> **截止日期：2026-05-21**（Phase 1 结束，功能范围冻结）
> **决策人**：全员

## 决策问题

MVP（最小可行产品）需要包含哪些功能？5/21 后新需求一律进 backlog。

## 已实现功能（Phase 1 提前完成）

以下功能已在 Phase 1 期间实现：

### 角色系统 ✅
- 5 步角色创建向导
- 6 种职业、22 种技能
- 4 个预设角色（快速开始）
- 随机角色名生成
- COC 7th Edition 属性体系

### AI DM 系统 ✅
- 多 Provider 支持（Anthropic / OpenAI / MiMo / 自定义）
- 严格 JSON 输出格式
- 上下文管理（12 轮对话限制）
- D100 技能检定集成
- 推荐行动建议系统

### 存档系统 ✅
- 多槽位存档/读档
- 继续游戏弹窗
- 存档删除
- API 配置持久化

### 基础设施 ✅
- 内置错误日志系统
- API Key 安全管理（config.local.js）
- API Key 历史记录

### 剧本内容 ✅
- 「雾中消逝」完整剧本框架（5 场景、8 线索物品、NPC 体系）

## 待确认是否纳入 MVP

以下功能需在 5/21 前决定是否纳入 MVP 范围：

- [ ] 背景音乐/音效系统
- [ ] 角色头像/场景插图（美术资产）
- [ ] 多剧本切换
- [ ] 多人协作（多客户端同步）
- [ ] 战斗系统（详细回合制）

## 已明确不在 MVP 范围

- 后端服务器
- 用户账号系统
- 在线多人实时联机
- 多语言支持

## 决策原则

- 5/21 后新需求一律进 backlog
- 宁少勿烂，确保已有功能稳定可用
- 优先保障核心循环（AI 叙述 -> 玩家行动 -> 检定 -> 结果）的流畅度
