# 原始资料摘要：AI 跑团游戏项目推进方案

## 核心内容提炼

### 项目定位
轻量级 Web 形式，AI 驱动叙事，四人团队，32 天内完成 MVP。

### 四个阶段

| 阶段 | 时间 | 核心目标 |
|------|------|----------|
| Phase 1 · 立项与设计 | 05/14–05/21（8天） | 锁定玩法、搭仓库、跑通 AI Demo |
| Phase 2 · 核心玩法开发 | 05/22–06/01（11天） | 主界面、角色创建、AI 接入、UI 组件 |
| Phase 3 · 内容填充 | 06/02–06/08（7天） | 剧本补全、存档、音效、联调 |
| Phase 4 · 测试与打磨 | 06/09–06/15（7天，含 Buffer） | 全员测试、Bug 修复、部署上线 |

### 协作规范要点
- 每周一次对齐会（30 分钟，来前准备状态更新）
- 唐龙翔负责 `/prompts` 文件编写，代码通过 PR 提交
- 茉莉统一命名：`scene_xxx.png` / `avatar_xxx.png`
- 提示词版本化：每次修改走 Git commit，新版本新建文件

### 风险预案
- 🔴 AI 输出不稳定 → 提示词版本化 + 兜底逻辑
- 🟡 剧本量跟不上 → 宁少勿烂，backlog 制
- 🟡 技术范围蔓延 → 5/21 后新需求一律进 backlog
- 🟢 API 费用 → 测试用便宜模型，加请求限流
