# 核心玩法循环

## 概述

游戏采用经典 TRPG 叙事循环：AI 描述场景 -> 玩家声明行动 -> AI 判定结果 -> 推进剧情。基于 COC 第七版规则，结合 D100 技能检定系统。

## 主循环流程

```
┌─────────────────────────────────────────────────┐
│                   游戏主循环                      │
│                                                  │
│  AI 叙述场景 ──> 玩家输入行动 ──> AI 处理行动     │
│       ↑                              │           │
│       │              ┌───────────────┼──────┐    │
│       │              │               ↓      │    │
│       │         需要检定？──是──> D100 投骰   │    │
│       │              │               │      │    │
│       │              否              ↓      │    │
│       │              │         检定结果回传   │    │
│       │              │               │      │    │
│       │              ↓               ↓      │    │
│       └──── AI 叙述结果（含状态更新）────────┘    │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 游戏模式

### 合作模式（Together）

所有玩家在同一场景中协作行动。

1. AI 描述当前场景
2. 所有玩家同时输入行动声明（声明池 declarations）
3. AI 综合所有声明，统一解决并叙述结果
4. 更新游戏状态

### 分离模式（Split）

玩家可分散到不同场景独立行动。

1. 各玩家位置由 `playerLocations` 追踪
2. 按玩家顺序轮流行动（`currentSplitPlayer`）
3. AI 根据玩家所在场景生成对应叙事
4. 各场景独立推进

## 技能检定流程

详见 [[entities/ai_dm]] 技能检定部分。

### 检定难度等级

| 难度 | 目标值 |
|------|--------|
| 普通 | <= 技能值 |
| 困难 | <= 技能值/2 |
| 极难 | <= 技能值/5 |

### 检定结果

| 结果 | 条件 | 说明 |
|------|------|------|
| 大成功 | 骰值 <= 技能值/5 | 极为出色的成功 |
| 困难成功 | 骰值 <= 技能值/2 | 超出预期的成功 |
| 成功 | 骰值 <= 技能值 | 正常完成 |
| 失败 | 骰值 > 技能值 | 未能完成 |
| 大失败 | 骰值 96-100 | 灾难性失败 |

## AI 推荐行动

每次 AI 响应后，会提供 3 个推荐行动（`playerChoices`），玩家可：
- 点击推荐行动快速输入
- 自由输入任意行动文本

## 状态更新

AI 每次响应可附带状态更新（`stateUpdate`）：
- HP 变化（受伤/治疗）
- SAN 变化（理智损失/恢复）
- 故事标记（flags）变化（触发剧情节点）

## 剧本模块：「雾中消逝」

当前内置剧本，设定于 1920 年 7 月 13 日，伦敦。

### 场景（5 个）

| ID | 场景 | 说明 |
|----|------|------|
| S01 | 莫尔宅邸 | 起始点，收集线索 |
| S02 | 警察局 | 审问蒙特利尔局长 |
| S03 | 老赫特酒馆 | 寻找"老鼠"的信息 |
| S04 | 卡森药房 | 核心地点，遭遇深潜者 |
| S05 | 泰晤士河港 / 安福丸 | 最终决战 |

### 线索物品（8 个）

笔记、照片、地图、雪茄等，散布在各场景中。
