# 团队成员

## 成员概览

| 成员 | 角色 | 核心职责 |
|------|------|----------|
| **Robert** | 技术架构 | 仓库管理、基础架构搭建、环境配置、部署上线、PR review |
| **唐龙翔** | 开发 / 策划 | 游戏前端主力开发、世界观与剧情框架、提示词编写 |
| **刘晓** | 玩法策划 | 跑团规则设计、剧本内容、AI 提示词工程（熟悉 TRPG） |
| **茉莉** | 美术 | UI 风格定调、角色头像、场景插图、界面视觉资产 |

## 各成员详情

### Robert · 技术架构
- **主要输出**：Git 仓库、基础设施、CI/部署配置
- **协作方式**：审核唐龙翔提交的 PR，提供技术支持
- **不负责**：日常游戏功能开发（由唐龙翔主导）

### 唐龙翔 · 开发 / 策划
- **主要输出**：游戏前端代码（`src/`）、世界观文档、`/prompts` 提示词文件
- **协作方式**：通过 PR 提交代码，Robert 审核合并
- **注意**：Vibe Coding 风格，需要 Robert 在架构层面给出清晰接口

### 刘晓 · 玩法策划
- **主要输出**：玩法一页纸、剧本 ch.1、提示词 v1、规则文档（`docs/`）
- **关键任务**：2026-05-18 前协助锁定 AI 角色定位
- **专业背景**：熟悉 TRPG，是提示词工程的主力

### 茉莉 · 美术
- **主要输出**：UI 风格参考板、核心视觉资产（`assets/`）
- **当前资源规范**：场景资源放在 `assets/scenes/`，NPC 立绘放在 `assets/npcs/`
- **交付建议**：场景图优先 16:9，NPC 立绘优先透明背景 PNG；临时大体积 GIF 后续替换为更可控资源
